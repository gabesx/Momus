# BB-MIG-01/02 Design — Bulk copy + reconciliation

**Date:** 2026-07-12  
**Status:** Approved for implementation (Approach A)

## Goal

Byte-copy legacy QARATMS MySQL tables into Momus Postgres, then reconcile counts and checksums (PRD §14 BB-MIG-01/02). Do **not** resync from Jira for history.

## Source of truth (local)

| | Value |
|---|---|
| Host | `127.0.0.1:3307` (docker `mysqldb`) |
| Database / user | `qara` / `qara` (app container env; host `.env` `tms`/`3309` is stale) |
| Target | Local Supabase `postgresql://postgres:postgres@127.0.0.1:54422/postgres` |
| QARATMS UI | http://localhost/performance |

Baseline (PRD §11.1 / live MySQL): **4653** rows — 2022:753 · 2023:1458 · 2024:912 · 2025:913 · 2026:617.

## Scope

**In:** `bug_budget`, `bug_budget_sync_runs`, `cron_schedules` row `bug_budget_sync`.  
**Out:** settings / multipliers / token (BB-MIG-03), parallel-run diffs (BB-MIG-04).

## Approach

TypeScript CLI package `@momus/migration`:

1. Read MySQL in batches → transform (JSON, bool, timestamps) → upsert/insert Postgres.
2. Preserve `bug_budget.id` / `sync_runs.id`; reset sequences after.
3. `sync_runs.requested_by`: use id if present in `users`, else map to fallback user (id 1 or `automated@system`).
4. Cron: upsert on `name = bug_budget_sync`.
5. Reconcile: `COUNT(*)`, per-year vs MySQL + §11.1, SHA-256 over sorted `(jira_key, updated_date, is_open)`.

## Secrets

Env only (`LEGACY_MYSQL_*`, `TARGET_DATABASE_URL`). Never commit passwords.
